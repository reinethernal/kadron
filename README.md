# Kadron Bot

## Описание проекта

Телеграм-бот для управления опросами и взаимодействия в группах. Проект
построен на [aiogram](https://github.com/aiogram/aiogram) и использует
архитектуру плагинов для расширения функциональности. Основные возможности
включают создание и экспорт опросов, тестовый режим, шаблоны опросов и управление правами доступа. Базовые функции аналитики реализованы на уровне базы данных и
используются для сбора статистики.

## Требования

- Python 3.10 или новее
- `pip` для установки зависимостей
- `aiogram` версии 3.x

Установите необходимые пакеты (файл `requirements.txt` также включает пакет
`packaging`, используемый для логики проверки версии aiogram):

```bash
pip install -r requirements.txt
```

## Переменные окружения

Скопируйте `example.env` в `.env` и укажите свои значения:

```bash
cp example.env .env
```

Переменные:

| Имя | Описание |
| --- | --- |
| `BOT_TOKEN` | Токен, полученный от BotFather |
| `DATABASE` | Путь к файлу базы SQLite |
| `ADMIN_IDS` | ID администраторов Telegram через запятую. Допустимы формы `123,456` и `[123,456]`. При первом запуске сохраняется в `bot_data.json` |
| `ENABLE_LOGGING` | Включить дополнительное логирование (`True` или `False`). Значение `False` полностью отключает вывод сообщений |
| `LOGGING_LEVEL` | Уровень логирования (например, `DEBUG` или `INFO`) |
| `ENABLE_CAPTCHA` | Использовать капчу для новых участников групп |
| `CAPTCHA_TIMEOUT` | Время ожидания капчи в минутах |
| `MAX_WARNINGS` | Максимальное число предупреждений перед действием |
| `ENABLE_INACTIVE_CLEANUP` | Удалять пользователей при длительной неактивности (`True` или `False`) |
| `INACTIVITY_DAYS` | Количество дней неактивности до удаления (используется при включённой очистке) |
| `WELCOME_MESSAGE` | Приветственное сообщение по умолчанию |
| `PLUGIN_DIR` | Путь к каталогу плагинов (по умолчанию используются `plugins_admin` и `plugins_surveys`) |
| `ADMIN_PLUGIN_DIR` | Путь к каталогу административных плагинов |
| `SURVEY_PLUGIN_DIR` | Путь к каталогу плагинов опросов |

При загрузке `storage_plugin` значение `ADMIN_IDS` из окружения сохраняется
в настройку `admin_ids` файла `bot_data.json`, если там ещё ничего нет.
Если позже список администраторов изменится, обновите его в `bot_data.json`
(см. `storage_plugin`).

Файл `bot_data.json` создаётся во время работы бота и хранит его состояние. Он не должен попадать в систему контроля версий.

### Отладочное логирование

Чтобы увидеть подробные сообщения в консоли, установите в файле `.env` переменную
`LOGGING_LEVEL` в значение `DEBUG` и перезапустите бот.
Чтобы полностью отключить логирование, задайте `ENABLE_LOGGING=False`.

## Запуск бота

После настройки `.env` бот запускается так:

```bash
cp example.env .env
# заполните BOT_TOKEN и ADMIN_IDS
python main.py
# Отправьте боту /start. Если ваш ID указан в ADMIN_IDS,
# под приветственным сообщением появится кнопка "Админ меню".
# Нажмите её, чтобы открыть главное меню без ввода команд.
```

Важно запускать `main.py` из корня репозитория. Если стартовать бот из другого каталога
(например, указав абсолютный путь к скрипту), менеджер плагинов может не найти
плагины в папках `plugins_admin` и `plugins_surveys`. Для нестандартных случаев можно задать путь к каталогу
плагинов через переменную окружения `PLUGIN_DIR` или задать каталоги отдельно
через `ADMIN_PLUGIN_DIR` и `SURVEY_PLUGIN_DIR`:

```bash
ADMIN_PLUGIN_DIR=/path/to/admin \
SURVEY_PLUGIN_DIR=/path/to/surveys \
python /absolute/path/to/main.py
```

Начиная с версии с относительными импортами, пакет с плагинами может называться как угодно
— достаточно передать корректный путь через `PLUGIN_DIR`.

Административные функции реализованы плагином `admin_menu_plugin`.
Этот плагин должен загружаться через `PluginManager` вместе со всеми
зависимостями, иначе инициализация завершится ошибкой.

## Запуск тестов

Перед запуском тестов установите зависимости из `requirements.txt` вручную или через скрипт:

```bash
pip install -r requirements.txt
# или
./scripts/setup_tests.sh
```

После установки запустите:

```bash
pytest
```

Тесты предполагают, что все перечисленные в `requirements.txt` пакеты уже
установлены.

Если установлено несколько версий Python, используйте соответствующий исполняемый файл (например, `python3.10 main.py`).

## Проверка стиля

В проекте используется `flake8` и `black` с длиной строки 88 символов.
Запустите форматирование и проверку так:

```bash
black . --line-length 88
flake8
```

## Архитектура плагинов

Плагины расположены в каталогах `plugins_admin` и `plugins_surveys`. `plugin_manager.py` автоматически загружает каждый файл с окончанием `_plugin.py`.
Функции планирования опросов реализованы отдельным `scheduler_plugin` в каталоге `plugins_surveys`.

Подробности создания собственных плагинов описаны в [CONTRIBUTING.md](CONTRIBUTING.md).

Пример использования:

```python
from aiogram import Dispatcher, Router
from plugin_manager import PluginManager

# dp — экземпляр Dispatcher
router = Router()
pm = PluginManager(dp, router=router)
await pm.load_plugins()
await pm.setup_bot_commands(bot)
```

При загрузке менеджер пытается инициализировать каждый найденный файл
плагина отдельно. Ошибки импорта или выполнения конкретного плагина
фиксируются в журнале и не мешают загрузке остальных. Исключение
`MissingRequiredPluginsError` возникает только после завершения цикла,
если какой-либо обязательный плагин так и не был загружен.

Чтобы отключить плагин, переименуйте или удалите его файл либо вызовите `unload_plugin` во время работы бота:

```python
await pm.unload_plugin('test_mode_plugin')  # имя без .py
```

Загрузить его снова можно вызовом `load_plugin('test_mode_plugin')`.

При изменении кода плагина его не обязательно отключать и включать вручную. Для
повторной загрузки существует метод `reload_plugin`:

```python
await pm.reload_plugin('test_mode_plugin')
```

Получить список загруженных плагинов можно методом `list_plugin_names()`:

```python
names = pm.list_plugin_names()
print(names)  # ['survey_plugin', 'admin_menu_plugin', 'plugin_list_plugin']
```
`plugin_list_plugin` добавляет команду `/plugins`, выводящую все загруженные плагины.

## Экспорт опросов

Для экспорта нажмите "Админ меню", затем "📈 Аналитика" и выберите
"Экспорт данных". Все прочие административные задачи доступны через
кнопки в том же меню.

## Шаблоны опросов

Плагин `survey_templates_plugin` упрощает повторное использование часто
встречающихся опросов. Шаблоны также управляются через меню администратора.

Шаблоны хранятся в файле `bot_data.json` вместе с остальными данными бота.

## Выбор групп для опросов

При создании или планировании опроса бот выводит список доступных групп из базы
данных (`core.db_manager.get_all_groups`). Отправьте ID нужных групп через
пробел, и они будут сохранены в поле `target_chats` создаваемого опроса.

## Не реализованные функции

В административном меню присутствуют пункты аналитики — «Статистика опросов»,
«Активность группы» и «Рейтинги». Эти разделы, а также часть опций в меню
«Настройки», пока не имеют реализации. При выборе таких пунктов бот отвечает
сообщением «Функция в разработке».

Тем не менее в модуле `core.db_manager` уже есть функции для получения
статистики:

- `get_survey_statistics(survey_id)` возвращает число участников, распределение
  ответов и среднее время прохождения опроса.
- `get_group_activity(chat_id, days)` подсчитывает сообщения, новых участников и
  реакции за заданный период.
- `get_user_ratings(chat_id)` собирает средние оценки пользователей и количество
  оставленных отзывов.

Эти функции можно использовать при разработке собственных инструментов
аналитики или при расширении текущего меню администратора.
